## Agent 硬規則

1. 未實際執行的命令不得描述為已完成。
2. 未出現在 log 或 result JSON 的測試不得判定通過。
3. 修改前必須有 Git checkpoint。
4. 不得混合功能修改、重構與格式化。
5. 測試工具錯誤時先修測試，不得依錯誤測試修改系統。
6. Engineer A 與 B 必須基於相同 base commit。
7. 提案階段不得修改檔案；實作階段只能修改指定 worktree。
8. 候選不得自行 commit。
9. Judge 不得修改程式。
10. 資訊不足時輸出 `INSUFFICIENT_EVIDENCE`。
11. 禁止修改 `CLAUDE.md` 與核心 `scripts/`；需要調整時由人類核准。
